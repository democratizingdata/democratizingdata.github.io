ACCESSIBILITY

Accessibility Statement

Democratizing Data is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone, and applying the relevant accessibility standards.

Last Updated: February 27, 2023

Our Commitment

Democratizing Data is committed to ensuring that our website is accessible to the widest possible audience, regardless of technology or ability. We aim to comply with all applicable standards and guidelines to ensure that our website can be used by everyone.

We believe that accessibility is not just about compliance—it's about creating an inclusive experience that empowers all users to access and benefit from data democratization tools and resources.

Conformance Status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA.

Democratizing Data is partially conformant with WCAG 2.1 Level AA. "Partially conformant" means that some parts of the content do not fully conform to the accessibility standard.

• Implemented

  • Semantic HTML structure
  • Keyboard navigation support
  • Focus indicators
  • Alternative text for images
  • Proper heading hierarchy
  • Color contrast compliance
  • Screen reader optimization
  • ARIA labels enhancement

⚠ In Progress

  • Form accessibility improvements
  • Video captions and transcripts

Compatibility with Browsers and Assistive Technology

Democratizing Data is designed to be compatible with the following assistive technologies:

  • Screen readers: NVDA, JAWS, VoiceOver, TalkBack
  • Voice recognition software: Dragon NaturallySpeaking
  • Keyboard navigation: Full keyboard accessibility
  • Browser zoom: Up to 200% magnification
  • High contrast modes: Windows High Contrast, macOS Increase Contrast

Supported Browsers

Our website is tested and optimized for:

  • Chrome (latest version)
  • Firefox (latest version)
  • Safari (latest version)
  • Edge (latest version)

Technical Specifications

Accessibility of Democratizing Data relies on the following technologies to work with the particular combination of web browser and any assistive technologies or plugins installed on your computer:

  • HTML5: Semantic markup and ARIA attributes
  • CSS3: Responsive design and accessibility features
  • JavaScript: Progressive enhancement and keyboard support
  • SVG: Scalable graphics with alternative text

These technologies are relied upon for conformance with the accessibility standards used.

Known Limitations

Despite our best efforts to ensure accessibility of Democratizing Data, there may be some limitations. Below is a description of known limitations, and potential solutions:

  • Third-party content: Some embedded content from external sources may not be fully accessible. We are working with providers to improve this.
  • Complex data visualizations: Some charts and graphs may require alternative text descriptions. We provide data tables as alternatives where possible.
  • PDF documents: Some older PDF documents may not be fully accessible. We are working to remediate these files.

If you encounter any of these limitations, please contact us for assistance or alternative formats.

Assessment Approach

Democratizing Data assessed the accessibility of our website through the following approaches:

  • Self-evaluation: Internal accessibility audits using automated tools and manual testing
  • User testing: Feedback from users with disabilities
  • Expert review: Consultation with accessibility specialists
  • Automated testing: Regular scans using accessibility testing tools

Feedback and Contact Information

We encourage users who encounter accessibility barriers by submitting an inquiry.

We aim to respond to accessibility feedback within 2 business days and to propose a solution within 5 business days.

When contacting us about accessibility, please include:

  • The web page or feature you were trying to access
  • A description of the problem you encountered
  • The assistive technology you were using (if applicable)
  • Your browser and operating system

Formal Complaints

If you are not satisfied with our response to your accessibility concern, you may file a formal complaint. We are committed to working with you to resolve any accessibility issues.

For formal complaints, you may also contact relevant authorities in your jurisdiction regarding digital accessibility requirements.
